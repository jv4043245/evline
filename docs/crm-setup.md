# EVLine CRM setup

Цель CRM: связать рекламу, заявку, заказ, доставку, выручку и маржу в одну цепочку.

## Что уже делает система

1. Формы сайта отправляют заявку в `/api/leads`.
2. Заявка сохраняется в `leads` и автоматически создает запись в `orders`.
3. В заказ попадают контакты, авто, VIN, запчасть/услуга, UTM, `gclid`, `fbclid`, landing page и referrer.
4. Менеджер ведет заказ в `/admin/`: меняет статус, добавляет цену, себестоимость, доставку, трек и заметки.
5. При смене статуса создается событие в истории заказа.
6. Если у клиента привязан Telegram `chat_id`, уведомление отправляется автоматически.
7. Если `chat_id` еще нет, уведомление остается в истории и его можно повторить после привязки.

## Переменные Cloudflare Pages

Обязательные:

- `ADMIN_TOKEN` - резервный токен для админки, если Cloudflare Access еще не включен.
- `TELEGRAM_BOT_TOKEN` - token бота, который отправляет уведомления.

Желательные:

- `TELEGRAM_PARTS_CHAT_ID` - внутренний чат заявок по запчастям.
- `TELEGRAM_TECH_CHAT_ID` - внутренний чат заявок по программированию BYD.
- `TELEGRAM_CHAT_ID` - общий fallback, если отдельные чаты не заданы.
- `TELEGRAM_WEBHOOK_SECRET` - secret token для Telegram webhook.

## Telegram webhook

Webhook endpoint:

```text
https://evline.com.ua/api/telegram/webhook
```

Команда установки webhook:

```bash
curl -X POST "https://api.telegram.org/bot$TELEGRAM_BOT_TOKEN/setWebhook" \
  -H "content-type: application/json" \
  -d '{"url":"https://evline.com.ua/api/telegram/webhook","secret_token":"CHANGE_ME"}'
```

Команда привязки клиента к заказу:

```text
/start order_<order_id>
```

Эта команда показана в карточке заказа. После того как клиент отправит ее боту, CRM запишет `telegram_chat_id` в заказ и карточку клиента.

Если статус поменяли до привязки клиента к боту, уведомление останется в истории со статусом `skipped`. После привязки CRM автоматически попробует отправить последнее неотправленное уведомление. В карточке заказа также есть ручной повтор для неотправленных уведомлений.

## Накат базы

Локальная проверка миграций:

```bash
sqlite3 /tmp/evline-crm.sqlite < migrations/0001_crm.sql
sqlite3 /tmp/evline-crm.sqlite < migrations/0002_orders_tracking.sql
```

Накат на Cloudflare D1:

```bash
npx wrangler d1 migrations apply evline-crm --remote
```

Если Wrangler недоступен, SQL из `migrations/*.sql` можно выполнить в Cloudflare D1 Console. В таком случае команды лучше выполнять отдельными statement'ами, а не одним большим полотном.

## Трекинг доставки

Автоматический трекинг сейчас включен для Meest China:

- карточка заказа хранит последний статус перевозчика, место, дату статуса, время последней проверки и расчетную дату доставки;
- история событий перевозчика пишется в `order_tracking_events`;
- кнопка `Оновити трекінг` в карточке заказа подтягивает свежие события Meest;
- endpoint `/api/admin/tracking/sync` может проверять открытые Meest-заказы пачкой;
- первый запуск по уже существующему треку сохраняет базовый статус без клиентского уведомления, следующие изменения создают событие и уведомление в Telegram, если клиент привязан к боту.

Ukr-China пока ведется вручную: их публичный трекинг требует телефон/SMS, поэтому для автоматизации нужен официальный API, webhook или выгрузка от перевозчика.

## Статусы заказа

- `new` - заявка получена.
- `accepted` - менеджер принял в работу.
- `proposal_sent` - предложение отправлено клиенту.
- `paid` - заказ оплачен.
- `sourcing_china` - ищем или заказываем в Китае.
- `china_warehouse` - заказ на складе в Китае.
- `left_china` - заказ выехал из Китая.
- `in_ukraine` - заказ в Украине.
- `ready_for_pickup` - готово к выдаче/отправке.
- `completed` - завершено.
- `canceled` - отменено.

## Что менеджер обязан заполнить для аналитики

Минимум для каждого заказа:

- телефон клиента;
- авто/модель;
- VIN, если есть;
- запчасть или услугу;
- статус заказа;
- выручку;
- закупку;
- доставку;
- рекламную стоимость заказа, если известна;
- причину отмены, если заказ отменен.

Для логистики:

- перевозчик;
- трек-номер;
- ссылка на трек;
- статус доставки.

## Как считать эффективность

CRM считает:

- заявки;
- активные и завершенные заказы;
- выручку;
- валовую маржу;
- CPL;
- ROAS;
- profit ROAS;
- эффективность источников и кампаний.

Чтобы эти цифры были честными, в блоке “Витрати на рекламу” нужно регулярно вносить расходы Google Ads/Facebook по датам и кампаниям.
